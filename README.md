# MVC_T2_Y2_09-02-2025
เลือกทำข้อที่ 2 :
  Model ทำหน้าที่ในการเก็บข้อมูล เเละการคำนวณต่างๆ
    ArmorDatabase : ทำหน้าที่ในการเป็น Database จำลอง เพื่อเก็บ Armor ชุดเกราะไว้  50 ตัว
      โดยจะมีชุดเกราะเเต่ละประเภท โดยจำลองให้เป็นรหัส 100000-100009 ชุดเกราะทรงพลัง(Powerful) , 200010-2000029 ชุดเกราะ ปกปิดตัวตน(Conceal), 300030-300049 เป็นชุดเกราะ Stralth (รอบเร้น)
    Armor : เก็บรหัสชุดเกราะ ประเภท เเละ พลังป้องกันไว้ มีส่วนของ ArmorCheck เอาไว้เเยกสำหรับเช็คเงื่อนไขตามประเภทของชุดเกราะ 
    ArmorCheck : เป็นต้นเเบบสำหรับเก็บว่าควรคิดคำนวณคุณภาพ ของเเต่ละชุดเกราะอย่างไป ตามประเภท โดยจะมี Powerful/Conceal/Stralth ArmorCheck เอาไว้เช็คเงื่อนไขดังกล่าว 
    
  View ทำหน้าที่ในการเเสดงผลข้อมูลให้ผู้ใข้ มีการ ติดต่อสื่อสารผู้ใช้    
    view : จะเเสดงเลขให้ผู้ใช้กรอก เลขรหัสของ ArmorID ว่าต้องการเลขใด มีปุ่มสำหรับกดส่งเลขรหัสเพื่อส่งไปยัง controller เพื่อจัดการต่อไป เเละ ยังใช้รับข้อความต่างๆ หรือ Event ที่ controller ส่งมาตามเงื่อนไข เพื่อเเสดงผลอีกด้วย

  Controller : ทำหน้าที่ในการประสานงานระหว่าง View & Model 
    ทำหน้าที่ : รับ Input จาก View ตาม ID โดยต้องกดปุ่ม submit โดยหน้่าที่นี้จะเป็นหน้าที่ของ submitButtonController  นำมาเช็คตรวจสอบว่า Input ในเบื้องต้นน้นถูกต้องไหม จาก isValidArmorID เเล้วจะทำการส่งไปยัง Model เพื่อให้หาตามเลขชุดเกราะนั้น หากมีข้อผิดพลาดก็จะให้ส่งข้อความไปหาฝ่าย view ว่าเกิดข้อผดพลาดเเบบใด เเล้วถ้าหากว่า เจอข้อมูลก็จะทำต่อโดย ส่งให้ Model ไปดู quality ให้ หากชุดเกราะไม่ตรงตามเงื่อนไขจะให้ view บอกว่า เกราะยังไม่พร้อมใช้ เเละ พร้อมให้เเสดงปุ่ม repair เพื่อซ่อมชุด ซึ่งถูกควบคุมด้วย RepairButtonContorller หากผู้ใช้กดปุ่มจะทำการให้ model ไปอัพเดตค่าชุดเกราะมาใหม่ เเล้วก็จะเอาค่าชุดเกราะไปให้ view เพื่อบอกว่า ชุดเกราะอัพเกรดค่าที่ถูกต้อง ถ้า ชุดเกราะตรงตามเงื่อนไข ก็จะให้ view บอกผู้ใช้ว่า ใช้ได้เเล้ว  เเล้วถ้าผู้ใช้ต้องการรีเซ็ตค่า ให้กด clear (Clear Controller ทำงานส่วนนี้) เพื่อหาชุดเกราะตัวใหม่ที่ผู้ใช้สนใจต่อไปครับบ
